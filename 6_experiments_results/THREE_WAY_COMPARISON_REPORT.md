# 三路对比实验结果汇总

## 实验概述

| 项目 | 详情 |
|------|------|
| **数据集** | 500条验证集样本 (Amazon Product Reviews) |
| **类别分布** | Negative: 159 (31.8%), Neutral: 201 (40.2%), Positive: 140 (28.0%) |
| **对比方法** | ① DeepSeek V3.2 (Cloud API) ② Qwen3-4B (Local LLM) ③ SVM+TF-IDF (Traditional) |

| 方法 | 准确率 | 宏平均P | 宏平均R | 宏平均F1 |

## 关键发现

### 1. 本地LLM达到云端96%性能

```
性能差距: 100% - 95.8% = 4.2%
```

使用 **动态温度蒸馏**（温度范围1.5-3.0），本地4-bit量化的Qwen3-4B模型达到了云端API **95.8%** 的准确率，仅落后 **4.2个百分点**。

**训练配置**:
- 模型: unsloth/Qwen3-4B-unsloth-bnb-4bit (4-bit量化)
- 硬件: RTX 3070 Ti Laptop (8GB VRAM)
- 训练时间: ~55分钟 (3 epochs)
- 蒸馏数据: 8,500条DeepSeek-R1软标注样本

### 2. 中性类识别是主要挑战

从混淆矩阵可以看出，所有错误分类都集中在 **中性类(1)**:

| 模型 | 主要错误类型 |
|------|-------------|
| Qwen3-4B | 6条负面→中性, 5条中性→正面, 8条正面→中性 |
| SVM | 大量混淆，中性类最差(F1=55.87%) |

**结论**: 中性评论（混合情感/轻度不满）是最难识别的类别。

### 3. 传统方法与现代LLM差距巨大

```
性能提升: 95.8% / 59.0% = 1.62x (62%相对提升)
绝对提升: 95.8% - 59.0% = 36.8个百分点
```

SVM+TF-IDF在3分类任务上表现不佳，主要原因是：
- 无法捕捉长距离语义依赖
- 对讽刺、反语等复杂表达理解有限
- 特征稀疏（TF-IDF）缺乏语义深度

---

## 关于DeepSeek V3.2的100%准确率

**重要说明**: DeepSeek V3.2显示100%准确率是因为这500条验证样本的**软标签本身就是由它生成的**。这不是一个公平的对比，而是：

1. **自评估**: 模型在评估自己的输出
2. **参考基准**: 展示"理论上限"性能
3. **蒸馏目标**: 本地模型学习的软标签来源

**更公平的对比方式**:
- 需要人工标注500条作为金标准
- 或使用不同模型（如Claude API）进行交叉验证

## 实验意义

### 知识蒸馏的有效性

| 指标 | 数值 |
|------|------|
| 教师模型 (DeepSeek V3.2) | 云端API，体积庞大 |
| 学生模型 (Qwen3-4B) | 本地4-bit量化，仅4.4GB显存 |
| 学生达到教师性能 | **95.8%** (仅落后4.2%) |
| 推理成本降低 | ~10倍 (API调用 vs 本地推理) |
| 隐私保护 | 数据无需上传云端 |

### 动态温度策略的价值

```
固定温度 T=2.0 基线: 86.50%
动态温度 T=1.5-3.0:    95.80%
                        ───────
提升:                   +9.30%
```

**动态温度策略**（基于EMNLP 2024成果）根据样本置信度自适应调整蒸馏温度：
- 高置信度(>0.9): T=1.5，保留锐利分布
- 中等置信度(0.6-0.9): T=2.0，标准平滑
- 低置信度(<0.6): T=2.5-3.0，更强平滑
